# Theoretical Background — Strategy Execution & KPI Management

This folder provides the theoretical and conceptual foundation for understanding the business domain that Rafed KPI is built upon. These documents are intended for new team members, stakeholders, and users who want to understand *why* the system works the way it does — not just *how* to use it.

## How These Topics Connect to Rafed KPI

```
Theory Topic                    →   How It Manifests in Rafed KPI
──────────────────────────────────────────────────────────────────────
Strategy Execution (01)         →   Entity type hierarchy (Pillars → Objectives → KPIs)
KPI Design (02)                 →   Entity fields: direction, period, unit, formula, target
Frameworks (03)                 →   Dashboard types: Executive, KPI Performance, Governance
Governance (04)                 →   Draft → Submit → Approve → Lock workflow
Health & RAG (05)               →   Health score gauges, color badges, Overview stat cards
```

## Recommended Reading Order

1. Start with **01 — Strategy Execution Fundamentals** to understand the big picture.
2. Read **02 — KPI Design** to understand how individual KPIs are structured.
3. Review **03 — Frameworks** to see how different organizations approach performance management.
4. Study **04 — Governance** to understand why the approval workflow exists.
5. Finish with **05 — Health & RAG** to understand how individual KPIs roll up into organizational scores.
