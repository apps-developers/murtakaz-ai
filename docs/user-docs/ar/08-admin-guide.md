# دليل المسؤول

<div dir="rtl">

يغطي هذا الدليل الوظائف الإدارية المتاحة للمستخدمين الذين يحملون دور **مسؤول المؤسسة (ADMIN)** أو **مسؤول المنصة (SUPER_ADMIN)**.

---

## الوصول إلى لوحة الإدارة

انتقل إلى `/<locale>/admin` عبر رابط **الإدارة** في الشريط الجانبي (يظهر فقط لأصحاب دور ADMIN وSUPER_ADMIN).

---

## أقسام لوحة الإدارة

### 1. إعدادات المؤسسة

تقع في أعلى صفحة الإدارة، وتتيح عرض وتعديل:

| الحقل | الوصف |
|-------|-------|
| **اسم المؤسسة** | الاسم المعروض للمؤسسة |
| **اسم المؤسسة (عربي)** | الاسم المعروض بالعربية |
| **النطاق (Domain)** | نطاق البريد الإلكتروني للمؤسسة (يُستخدم لتحديد نطاق المستخدمين) |
| **رابط الشعار** | رابط صورة شعار المؤسسة |
| **الرسالة** | رسالة المؤسسة (بالإنجليزية) |
| **الرسالة (عربي)** | رسالة المؤسسة بالعربية |
| **الرؤية** | رؤية المؤسسة (بالإنجليزية) |
| **الرؤية (عربي)** | رؤية المؤسسة بالعربية |
| **مستوى اعتماد مؤشرات الأداء** | الجهة التي تعتمد قيم مؤشرات الأداء: `MANAGER` أو `EXECUTIVE` أو `ADMIN` |
| **حد الأخضر (RAG)** | أدنى نسبة إنجاز تُصنَّف خضراء (الافتراضي: 75%) |
| **حد الأصفر (RAG)** | أدنى نسبة إنجاز تُصنَّف صفراء (الافتراضي: 50%). ما دون هذا الحد يُعدّ أحمر. |
| **جهات الاتصال** | معلومات التواصل للمؤسسة (JSON) |

> يؤثر تغيير **مستوى الاعتماد** فوراً على الدور الذي يرى الإرسالات في طابور الاعتمادات.

> يؤثر تغيير **حدود RAG** فوراً على جميع لوحات المتابعة وعروض الصحة عبر المؤسسة.

---

### 2. إدارة المستخدمين

انتقل إلى **الإدارة ← المستخدمون** (`/<locale>/admin/users`).

تعرض قائمة المستخدمين جميع مستخدمي المؤسسة مع:
- الاسم والبريد الإلكتروني
- شارة الدور
- المدير المباشر (إن وُجد)
- المسمى الوظيفي
- تاريخ الإنشاء

#### إنشاء مستخدم جديد

1. انقر على **+ مستخدم جديد** في صفحة المستخدمين.
2. أملأ البيانات:
   - **الاسم** (مطلوب)
   - **البريد الإلكتروني** (مطلوب، يجب أن يكون فريداً داخل المؤسسة)
   - **الدور** (مطلوب): `ADMIN` أو `EXECUTIVE` أو `MANAGER`
   - **المسمى الوظيفي** (اختياري)
   - **المدير المباشر** (اختياري): يربط هذا المستخدم بمديره في الهيكل التنظيمي
   - **كلمة المرور** (مطلوبة لتسجيل الدخول بالبريد الإلكتروني وكلمة المرور)
3. انقر على **حفظ**.

يمكن للمستخدم تسجيل الدخول فوراً ببيانات الاعتماد المُدخَلة.

#### تعديل مستخدم

1. من قائمة المستخدمين، انقر على اسم المستخدم أو أيقونة التعديل.
2. عدّل الحقول المطلوبة.
3. انقر على **حفظ**.

#### تغيير دور مستخدم

1. افتح صفحة تفاصيل المستخدم.
2. حدّث حقل **الدور**.
3. احفظ.

يسري الدور الجديد فور تحديث المستخدم للصفحة.

#### إلغاء تفعيل مستخدم / حذفه

يُطبَّق الحذف الناعم — يُخفى المستخدم من جميع عروض المؤسسة لكنه يظل محفوظاً لأغراض التدقيق. تواصل مع مسؤول المنصة (SUPER_ADMIN) إذا كان الحذف الدائم مطلوباً.

---

### 3. أنواع كيانات المؤسسة

تُعرِّف أنواع الكيانات اللبنات المرنة لنموذج الاستراتيجية في مؤسستك. تُدار على مستوى المؤسسة.

انتقل إلى **الإدارة ← المؤسسات ← [مؤسستك] ← أنواع الكيانات** (يُتاح عبر لوحة مسؤول المنصة أو عبر API).

| الحقل | الوصف |
|-------|-------|
| **الرمز (Code)** | معرّف فريد قصير يُستخدم في الروابط (مثل: `kpi`، `pillar`) |
| **الاسم** | الاسم المعروض بالإنجليزية |
| **الاسم (عربي)** | الاسم المعروض بالعربية |
| **ترتيب العرض** | يتحكم في الترتيب في قائمة التنقل الجانبية |

> رموز أنواع الكيانات ثابتة بعد إنشائها وارتباطها بالكيانات. تغيير حقلَي `name`/`nameAr` آمن في أي وقت.

---

### 4. سجل التدقيق

تتضمن صفحة الإدارة لوحة **سجل التدقيق** التي تعرض أبرز أحداث النظام الأخيرة:

- تحديثات أهداف مؤشرات الأداء
- إعادة تكليف الملكية
- تصعيد المخاطر

سجل التدقيق **غير قابل للتعديل** — لا يمكن حذف الإدخالات أو تعديلها. يعرض النسخة الحالية أبرز الأحداث الأخيرة. سيتوفر سجل تدقيق كامل قابل للبحث في إصدار مستقبلي.

---

### إعدادات مسؤول المنصة

انتقل إلى `/<locale>/super-admin/settings` لإدارة الإعدادات على مستوى النظام:

| الإعداد | الوصف |
|---------|-------|
| **مؤشرات الميزات (Feature Flags)** | تمكين/تعطيل ميزات النظام |
| **سمة الألوان** | تخصيص لون التميز الرئيسي للنظام |

#### مؤشرات الميزات (Feature Flags)

تتحكم هذه المؤشرات في توفر الميزات في النظام:

| الميزة | الوصف |
|--------|-------|
| **الميزات الذكية** | لوحة المساعد الذكي (AI Assistant) |
| **المخططات البيانية** | الميزات التفاعلية والرسوم البيانية المتقدمة |
| **الميزات المتقدمة** | الميزات التجريبية والمتقدمة |
| **سير عمل الاعتمادات** | نظام طابور الاعتمادات |
| **المرفقات** | إمكانية رفع الملفات والمرفقات |
| **لوحات المتابعة** | الوصول إلى لوحات المتابعة |
| **الإشعارات** | نظام الإشعارات والتنبيهات |
| **سجلات التدقيق** | تسجيل الأنشطة والتدقيق |

> **ملاحظة:** يتطلب تغيير الميزات صلاحيات SUPER_ADMIN ويؤثر على جميع المؤسسات في النظام.

#### سمة الألوان (Color Theme)

يمكن تخصيص لون التميز الرئيسي للنظام:

1. انتقل إلى **الإعدادات** في لوحة مسؤول المنصة.
2. اختر **سمة الألوان**.
3. اختر من الألوان المتاحة:
   - أزرق (Blue) — الافتراضي
   - أخضر زمردي (Emerald)
   - بنفسجي (Violet)
   - برتقالي (Orange)
   - زهري (Rose)
   - وغيرها...

> يتم تطبيق لون السمة فوراً على واجهة المستخدم لجميع المستخدمين.

## لوحة مسؤول المنصة (Super Admin)

لوحة **مسؤول المنصة** (`/<locale>/super-admin`) لا يمكن الوصول إليها إلا للمستخدمين الذين يحملون دور `SUPER_ADMIN`. توفر إدارة على مستوى المنصة:

### صلاحيات مسؤول المنصة

| الصلاحية | الوصف |
|---------|-------|
| **إدارة المؤسسات** | إنشاء المؤسسات وعرضها وتعديلها وحذفها |
| **إدارة المستخدمين عبر المؤسسات** | عرض وإدارة مستخدمي أي مؤسسة |
| **البيانات التجريبية (Seed)** | تحميل البيانات الأولية أو إعادة تعيينها لأي مؤسسة |
| **صحة النظام** | عرض مقاييس المنصة على مستوى النظام |

### إنشاء مؤسسة جديدة

1. انتقل إلى `/super-admin/organizations`.
2. انقر على **+ مؤسسة جديدة**.
3. أملأ حقلَي الاسم والنطاق والحقول الأخرى.
4. انقر على **حفظ**.

بعد الإنشاء، يمكنك إضافة مستخدمين وأنواع كيانات للمؤسسة.

### تحميل البيانات الأولية (Seeding)

لتعبئة مؤسسة جديدة ببيانات تجريبية/أولية بسرعة:

1. انتقل إلى `/super-admin/organizations/<orgId>/seed`.
2. اختر ملف بيانات أولية أو ارفع ملف JSON.
3. أكّد التحميل.

> **تحذير:** يؤدي التحميل إلى استبدال البيانات أو إلحاقها بالمؤسسة. راجع محتوى البيانات قبل التأكيد.

## المسؤوليات (تكليفات المستخدم بالكيانات)

انتقل إلى `/<locale>/responsibilities` لإدارة الكيانات التي يُكلَّف بها كل مستخدم.

تتحكم التكليفات في:
- هل يمكن للمستخدم ذو دور المدير رؤية كيان محدد؟
- هل يمكن للمستخدم إدخال قيم لذلك الكيان؟

### تكليف مستخدم بكيان

1. افتح صفحة تفاصيل الكيان.
2. انتقل إلى تبويب **التكليفات**.
3. انقر على **+ تكليف مستخدم**.
4. ابحث عن المستخدم وحدده.
5. أكّد.

أو استخدم واجهة إدارة التكليفات في صفحة المسؤوليات.

### إزالة تكليف

من تبويب التكليفات للكيان، انقر على إجراء الإزالة بجانب اسم المستخدم.

---

## مهام إدارية شائعة

### لماذا لا يستطيع مستخدم رؤية كيان معين؟

1. تأكد من **دور المستخدم** — المدير يرى الكيانات المكلَّف بها فقط.
2. تحقق من تبويب **التكليفات** للكيان — هل المستخدم مدرج فيه؟
3. إذا لم يكن كذلك، أضف تكليفاً لذلك المستخدم.

### إعادة تعيين اعتماد متوقف

1. افتح تبويب **القيم** للكيان.
2. ابحث عن القيمة في حالة SUBMITTED.
3. استخدم إجراء **الرفض** لإعادتها إلى DRAFT.
4. أشعر المُرسِل بإعادة الإدخال والإرسال.

### إضافة نوع مؤشر أداء جديد إلى الشريط الجانبي

1. انتقل إلى إعداد **أنواع الكيانات** للمؤسسة.
2. أضف نوع كيان جديداً بالرمز والاسم وترتيب العرض المطلوبة.
3. سيظهر النوع الجديد تلقائياً في قائمة التنقل الجانبية.

---

## ملاحظات الأمان

- تُخزَّن كلمات المرور على شكل **تجزئات bcrypt** — لا تُخزَّن أبداً بنص عادي.
- تنتهي صلاحية الجلسات بعد المدة المهيَّأة لها.
- ترتبط جميع تعديلات الكيانات بـ`orgId` للمستخدم المُصادَق عليه — يُمنع الوصول إلى بيانات المؤسسات الأخرى على مستوى الاستعلامات.
- سجلات التدقيق للإلحاق فقط ولا يمكن تعديلها.
