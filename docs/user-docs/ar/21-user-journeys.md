# رحلات المستخدم

<div dir="rtl">

دليل شامل لمسارات المستخدم في منصة  المؤشرات KPI، يوضح كيف يتنقل كل دور من الأدوار في النظام لتحقيق أهدافه.

---

## نظرة عامة على الأدوار

```mermaid
mindmap
  root(( المؤشرات KPI))
    التنفيذي
      الإشراف على الأداء
      اتخاذ القرارات
      مراجعة التقارير
    المدير
      متابعة الفريق
      إدخال البيانات
      تتبع المشاريع
    المسؤول
      إدارة المستخدمين
      تهيئة النظام
      الحوكمة
    المسؤول العام
      إدارة المنصة
      دعم المؤسسات
      الإعدادات العامة
```

---

## الرحلة أ — التنفيذي (Executive)

### 👤 ملف الشخصية

| | |
|:---|:---|
| **الدور** | EXECUTIVE |
| **الهدف** | الإشراف الاستراتيجي واتخاذ القرارات المبنية على البيانات |
| **الوقت في النظام** | 30-60 دقيقة يومياً |
| **الأولويات** | النظرة العامة → التقارير → الاعتمادات |

### مسار الرحلة

```mermaid
journey
  title رحلة التنفيذي اليومية
  section تسجيل الدخول
    فتح التطبيق: 5: التنفيذي
    مراجعة النظرة العامة: 4: التنفيذي
  section التحليل
    فتح التقارير: 4: التنفيذي
    مراجعة KPIs: 5: التنفيذي
    التفصيل في المخاطر: 3: التنفيذي
  section القرارات
    مراجعة الاعتمادات: 4: التنفيذي
    الموافقة/الرفض: 5: التنفيذي
```

### تدفق العمل

```mermaid
flowchart TD
    A[تسجيل الدخول] --> B{النظرة العامة}
    B -->|تحتاج اهتمام| C[لوحة المخاطر]
    B -->|أداء جيد| D[التقارير]
    
    C --> E[تفصيل في الكيان]
    E --> F[مراجعة المؤشرات]
    F --> G{قرار}
    G -->|اعتماد| H[الموافقة]
    G -->|رفض| I[إضافة تعليق]
    
    D --> J[التقرير التنفيذي]
    D --> K[محاذاة استراتيجية]
    D --> L[أداء KPIs]
    
    H --> M[إشعار للمعنيين]
    I --> M
    
    style B fill:#fef3c7,stroke:#f59e0b
    style G fill:#dbeafe,stroke:#3b82f6
    style H fill:#dcfce7,stroke:#22c55e
    style I fill:#fee2e2,stroke:#ef4444
```

## الرحلة ب — المدير (Manager)

### 👤 ملف الشخصية

| | |
|:---|:---|
| **الدور** | MANAGER |
| **الهدف** | تنفيذ الاستراتيجية عبر فريقي وضمان تحديث البيانات |
| **الوقت في النظام** | 1-2 ساعة يومياً |
| **الأولويات** | إدخال القيم → متابعة الفريق → الاعتمادات |

### مسار الرحلة

```mermaid
journey
  title رحلة المدير اليومية
  section الصباح
    مراجعة المهام: 5: المدير
    تحديث KPIs: 4: المدير
    إرسال للاعتماد: 4: المدير
  section المتابعة
    مراجعة فريقي: 5: المدير
    تذكير المرؤوسين: 3: المدير
  section التحليل
    فتح تقارير الفريق: 4: المدير
    تحديد المشاكل: 4: المدير
```

### تدفق العمل

```mermaid
flowchart LR
    subgraph الصباح[بدء اليوم]
        A[تسجيل الدخول] --> B[المسؤوليات]
        B --> C[الكيانات المُسندة]
    end
    
    subgraph الإدخال[إدخال البيانات]
        D[KPI للتحديث] --> E[إدخال القيمة]
        E --> F[حفظ كمسودة]
        F --> G{قيمة نهائية؟}
        G -->|نعم| H[إرسال للاعتماد]
        G -->|لا| I[متابعة لاحقاً]
    end
    
    subgraph المتابعة[متابعة الفريق]
        J[فريقي] --> K[حالة التحديث]
        K -->|متأخر| L[إرسال تذكير]
        K -->|محدث| M[تأكيد جيد]
    end
    
    C --> D
    H --> N[انتظار الاعتماد]
    C --> J
    
    style H fill:#dcfce7,stroke:#22c55e
    style L fill:#fef3c7,stroke:#f59e0b
```

### دورة حياة قيمة KPI

```mermaid
stateDiagram-v2
    [*] --> مسودة : إدخال قيمة
    مسودة --> مسودة : تحديث
    مسودة --> مرسلة : إرسال للاعتماد
    مرسلة --> معتمدة : موافقة
    مرسلة --> مسودة : رفض
    معتمدة --> مقفلة : نهاية الفترة
    معتمدة --> [*] : استخدام في التقارير
```

### الخطوات التفصيلية

| الخطوة | الصفحة | الإجراء | النتيجة |
|--------|--------|---------|---------|
| 1 | `/ar/responsibilities` | مراجعة تكليفاتي | معرفة ما يجب تحديثه |
| 2 | `/ar/entities/kpi` | اختيار KPI | فتح صفحة KPI |
| 3 | تبويب "القيم" | إدخال قيمة جديدة | حفظ كمسودة |
| 4 | نفس الصفحة | إرسال للاعتماد | انتظار الاعتماد |
| 5 | `/ar/dashboards/manager` | مراجعة أداء الفريق | متابعة التقدم |

### نصائح للمدير

> 💡 **تذكير:** ضع موعداً ثابتاً يومياً (مثلاً 9 صباحاً) لتحديث KPIs
> 
> 💡 **تنظيم:** صفح الأهمية - ابدأ بـ KPIs ذات الأولوية العالية

## الرحلة ج — المسؤول (Admin)

### 👤 ملف الشخصية

| | |
|:---|:---|
| **الدور** | ADMIN |
| **الهدف** | إدارة المؤسسة وضمان سير العمل بفعالية |
| **الوقت في النظام** | 2-4 ساعات أسبوعياً |
| **الأولويات** | إدارة المستخدمين → الهيكل التنظيمي → الحوكمة |

### خريطة الإدارة

```mermaid
mindmap
  root((المسؤول))
    إدارة المستخدمين
      إنشاء حسابات
      تعيين الأدوار
      تعيين المدراء
    الهيكل التنظيمي
      الإدارات
      الفرق
      العلاقات
    التكليفات
      تكليف المستخدمين
      الصلاحيات
    الإعدادات
      مستويات الاعتماد
      الألوان والشعار
      الميزات
```

### تدفق إعداد مؤسسة جديدة

```mermaid
flowchart TD
    A[إنشاء المؤسسة] --> B[إضافة الإدارات]
    B --> C[إنشاء المستخدمين]
    C --> D[تعيين المدراء]
    D --> E[تعيين أنواع الكيانات]
    E --> F[إنشاء الكيانات]
    F --> G[تكليف المستخدمين]
    G --> H[تهيئة لوحات المتابعة]
    H --> I[اختبار سير العمل]
    I --> J[نشر للمستخدمين]
    
    style A fill:#dbeafe,stroke:#3b82f6
    style J fill:#dcfce7,stroke:#22c55e
```

### الخطوات التفصيلية

| المهمة | الصفحة | الإجراء | النتيجة |
|--------|--------|---------|---------|
| إنشاء مستخدم | `/ar/admin/users` | جديد → إدخال البيانات → حفظ | مستخدم جاهز |
| إنشاء إدارة | `/ar/departments` | جديد → اسم → مدير | إدارة في الهيكل |
| تكليف مستخدم | `/ar/responsibilities` | اختر الكيان → اختر المستخدم | صلاحية محددة |
| تغيير إعداد | `/ar/organization` | تعديل → حفظ | تطبيق على النظام |

## الرحلة هـ — الاعتمادات (Approval Workflow)

### تدفق الموافقة

```mermaid
sequenceDiagram
    actor M as المُدخل
    actor A as المعتمد
    participant S as النظام
    participant DB as قاعدة البيانات
    
    M->>S: إرسال قيمة KPI
    S->>DB: حفظ كـ "مرسلة"
    S->>A: إشعار بالاعتماد
    
    alt موافقة
        A->>S: اعتماد القيمة
        S->>DB: تحديث لـ "معتمدة"
        S->>M: إشعار بالموافقة
    else رفض
        A->>S: رفض القيمة
        A->>S: إضافة تعليق
        S->>DB: إرجاع لـ "مسودة"
        S->>M: إشعار بالرفض
    end
```

### حالات الاعتماد

```mermaid
stateDiagram-v2
    [*] --> مسودة
    مسودة --> مرسلة : إرسال
    مرسلة --> معتمدة : موافقة
    مرسلة --> مسودة : رفض
    معتمدة --> مقفلة : انتهاء الفترة
    مقفلة --> [*] : أرشفة
    
    note right of مسودة
        قابلة للتعديل
        فقط المُدخل يراها
    end note
    
    note right of مرسلة
        مقفلة للتعديل
        إشعار للمعتمدين
    end note
    
    note right of معتمدة
        نهائية
        تُستخدم في التقارير
    end note
```

## الرحلة و — المسؤوليات (Responsibilities)

### طرق العرض

```mermaid
graph TB
    subgraph طرق_العرض["طرق عرض المسؤوليات"]
        A[حسب المستخدم] --> B[عرض كل مستخدم والكيانات المُسندة له]
        C[حسب الكيان] --> D[عرض كل كيان والمستخدمين المُسندين له]
    end
    
    subgraph الإجراءات["الإجراءات المتاحة"]
        E[تكليف جديد] --> F[اختيار مستخدمين + كيانات]
        G[إلغاء تكليف] --> H[إزالة صلاحية]
        I[تعديل دور] --> J[OWNER → CONTRIBUTOR → REVIEWER]
    end
    
    B --> E
    D --> E
    B --> G
    D --> G
```

### صلاحيات الأدوار

| الدور | إنشاء | قراءة | تحديث | حذف | اعتماد |
|-------|:-----:|:-----:|:-----:|:---:|:------:|
| OWNER | ✅ | ✅ | ✅ | ✅ | ✅ |
| CONTRIBUTOR | ✅ | ✅ | ✅ | ❌ | ❌ |
| REVIEWER | ❌ | ✅ | ❌ | ❌ | ✅ |

## نصائح عامة لجميع المستخدمين

### 🚀 اختصارات سريعة

```
Ctrl+K    → فتح البحث
Ctrl+/    → عرض اختصارات لوحة المفاتيح
?         → المساعدة السياقية
```

### 📱 التنقل بين الأقسام

```mermaid
flowchart LR
    A[النظرة العامة] -->|الكيانات| B[قائمة KPIs]
    B -->|KPI محدد| C[تفاصيل KPI]
    C -->|القيم| D[إدخال القيمة]
    D -->|إرسال| E[قائمة الاعتمادات]
    E -->|موافقة| F[تقرير مُحدث]
    F -->|رجوع| A
```

### ⚡ تحسين الإنتاجية

1. **استخدم الفلاتر** لتضييق نطاق البحث
2. **فعّل الإشعارات** للحصول على التنبيهات فوراً
3. **احفظ المفضلة** في المتصفح للوصول السريع
4. **استخدم التطبيق** على الجوال للمتابعة من أي مكان

## دعم المستخدم

### حل مشاكل شائعة

| المشكلة | الحل |
|---------|------|
| لا أجد KPI | تحقق من التكليفات في `/ar/responsibilities` |
| لا أستطيع الإرسال | تأكد من ملء جميع الحقول المطلوبة |
| لا أرى زر الموافقة | تحقق من أن لديك صلاحية الاعتماد |
