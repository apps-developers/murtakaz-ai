<div dir="rtl">

دليل شامل لإدارة البيانات، النسخ الاحتياطي، وإعادة ضبط النظام.

## هندسة تخزين البيانات

### قاعدة البيانات

يستخدم التطبيق **قاعدة بيانات PostgreSQL** مع Prisma ORM لجميع عمليات تخزين البيانات:

### مخطط قاعدة البيانات

الكيانات الرئيسية المخزنة:

| الكيان | الوصف |
|--------|-------|
| **المؤسسات (Organizations)** | بيانات المؤسسة متعددة المستأجرين |
| **المستخدمون (Users)** | حسابات المستخدمين مع الأدوار وعلاقات المدير |
| **الإدارات (Departments)** | الإدارات التنظيمية مع المدراء |
| **أنواع الكيانات (Entity Types)** | أنواع الكيانات القابلة للتهيئة (الركائز، الأهداف، مؤشرات الأداء، إلخ) |
| **الكيانات (Entities)** | كيانات استراتيجية خاصة بالمؤسسة |
| **تكليفات الكيانات (Entity Assignments)** | تعيينات المسؤوليات للمستخدمين بالكيانات |
| **طلبات الاعتماد (Approval Requests)** | طلبات سير عمل الحوكمة |
| **الإشعارات (Notifications)** | قائمة إشعارات المستخدم |
| **مفاتيح الميزات (Feature Flags)** | مفاتيح تبديل الميزات على مستوى النظام |

---

## تخزين الملفات

- الملفات المحملة من قبل المستخدمين تُخزَّن في `web/public/uploads/`
- شعارات المؤسسات والأصول تُخدم من `/uploads/`

---

## النسخ الاحتياطي والاسترداد

### نسخ قاعدة البيانات الاحتياطية

تُخزَّن ملفات النسخ الاحتياطي في `data/backups/`:

- `backup_*.sql` — نسخ كاملة لقاعدة البيانات
- `*_snapshot_*.json` — لقطات JSON لبيانات البذر الأولية

---

### إنشاء نسخة احتياطية

```bash
# من داخل مجلد web
npx prisma db seed --backup
```

أو استخدم سكربت النسخ الاحتياطي:

```bash
cd web && npx tsx prisma/backup-db.ts
```

## إجراءات إعادة الضبط

### إعادة ضبط جلسة المستخدم

1. انقر على صورة المستخدم في الشريط الجانبي
2. اختر **تسجيل الخروج**
3. أو استدعِ `POST /api/session/logout`

---

### إعادة ضبط قاعدة البيانات (للتطوير فقط)

⚠️ **تحذير**: سيحذف هذا جميع البيانات!

```bash
cd web

# إعادة ضبط قاعدة البيانات وتطبيق الترحيلات
npx prisma migrate reset

# بذر البيانات الأولية
npx prisma db seed
```

---

### الاسترداد من نسخة احتياطية

```bash
# الاسترداد من نسخة SQL احتياطية
psql -d database_name -f data/backups/backup_YYYYMMDD_HHMMSS.sql
```

## تكوين البيئة

### متغيرات البيئة المطلوبة

انظر إلى `web/.env.example`:

| المتغير | الوصف |
|---------|-------|
| `DATABASE_URL` | سلسلة اتصال PostgreSQL |
| `NEXTAUTH_SECRET` | مفتاح تشفير الجلسات |
| `NEXTAUTH_URL` | عنوان URL الأساسي للتطبيق |

---

### إعادة ضبط مفاتيح الميزات

تُخزَّن مفاتيح الميزات في قاعدة البيانات ويمكن إعادة ضبطها عبر:

- `/<locale>/super-admin/settings` — تبديل الميزات الفردية
- جدول قاعدة البيانات: `FeatureFlag`

---

## حالة جانب العميل (Client-Side State)

تُخزَّن تفضيلات واجهة المستخدم التالية في `localStorage` الخاص بالمتصفح:

| المفتاح | الوصف |
|---------|-------|
| `theme` | تفضيل الوضع الفاتح/الداكن |
| `locale` | تفضيل اللغة |
| `ai-chat-open` | حالة لوحة المساعد الذكي |
| `sidebar-pinned` | حالة تثبيت الشريط الجانبي |

⚠️ مسح بيانات المتصفح سيُعيد ضبط هذه التفضيلات ولكن **لن تؤثر** على بيانات الخادم.

| المهمة | التكرار | المسؤول |

## استكشاف الأخطاء الشائعة

### ❌ "Database connection failed"

**السبب:** قاعدة البيانات غير متوفرة أو تكوين `DATABASE_URL` غير صحيح.

**الحل:**
1. تحقق من تشغيل خادم PostgreSQL
2. تحقق من صحة سلسلة الاتصال في `.env`
3. تأكد من وجود قاعدة البيانات

### ❌ "Permission denied" عند النسخ الاحتياطي

**السبب:** صلاحيات غير كافية لكتابة في مجلد النسخ الاحتياطي.

**الحل:**
```bash
chmod 755 data/backups/
```

### ❌ "Feature flag not found"

**السبب:** مفتاح ميزة غير موجود في قاعدة البيانات.

**الحل:**
1. انتقل إلى `/<locale>/super-admin/settings`
2. أعد حفظ الإعدادات لإعادة إنشاء المفاتيح المفقودة
