# الكيانات ومؤشرات الأداء

<div dir="rtl">

**الكيانات** هي اللبنات الأساسية في المنصة. يُمثَّل كل مؤشر أداء أو مبادرة استراتيجية أو هدف أو ركيزة على شكل كيان. تحدد مؤسستك أنواع الكيانات الموجودة وأسماءها.

---

## فهم أنواع الكيانات

تُهيِّئ كل مؤسسة **أنواع الكيانات** الخاصة بها (مثل: "الركيزة الاستراتيجية"، "المبادرة"، "مؤشر الأداء"، "الهدف"). لكل نوع:

- **رمز (Code)** — معرّف قصير يُستخدم في الروابط (مثل: `/entities/kpi`)
- **اسم** — يُعرض باللغة الإنجليزية
- **اسم عربي** — يُعرض باللغة العربية
- **ترتيب العرض** — يتحكم في ترتيب ظهوره في قائمة التنقل

تُهيَّأ أنواع الكيانات من قِبَل المسؤول ولا يمكن للمستخدمين العاديين تعديلها.

---

## تصفح الكيانات

### صفحة قائمة الكيانات

انتقل إلى **الكيانات ← [اسم النوع]** في الشريط الجانبي، أو انتقل مباشرةً إلى:

```
/<locale>/entities/<entityTypeCode>
```

تعرض صفحة القائمة:
- عنوان الكيان (والعنوان العربي إن وُجد)
- **مقياس مؤشر الأداء** يُظهر قيمة الإنجاز الحالية مقابل الهدف
- اسم المالك
- شارة الحالة (`PLANNED` مخطط، `ACTIVE` نشط، `AT_RISK` في خطر، `COMPLETED` مكتمل)
- إجراءات سريعة: تعديل (للمسؤول فقط)، حذف (للمسؤول فقط)

**البحث**: استخدم مربع البحث في الأعلى للتصفية حسب العنوان.

**التصفح**: تُحمَّل ما يصل إلى 250 عنصراً في كل مرة؛ استخدم البحث لتضييق النتائج.

---

## صفحة تفاصيل الكيان

انقر على اسم أي كيان لفتح صفحة تفاصيله على:

```
/<locale>/entities/<entityTypeCode>/<entityId>
```

صفحة التفاصيل منظَّمة في تبويبات:

| التبويب | المحتوى |
|---------|---------|
| **النظرة العامة** | العنوان، الوصف، المالك، الحالة، الهدف، القيمة الأساسية، الوحدة، نوع الدورية، الصيغة الحسابية |
| **القيم** | إدخالات القيم التاريخية مع حالتها (مسودة / مُرسَل / معتمد / مقفل) |
| **المتغيرات** | متغيرات الإدخال المستخدمة في مؤشرات الأداء القائمة على الصيغ الحسابية (CALCULATED) |
| **التكليفات** | المستخدمون المكلَّفون بهذا الكيان |
| **المرفقات** | الملفات والروابط المرتبطة بهذا الكيان |

---

## شرح حقول الكيان

| الحقل | الوصف |
|-------|-------|
| **العنوان** | الاسم المعروض (بالإنجليزية) |
| **العنوان (عربي)** | الاسم المعروض (بالعربية) — اختياري |
| **الوصف** | نص حر لوصف الكيان |
| **المالك** | المستخدم المسؤول عن هذا الكيان |
| **الحالة** | `PLANNED` / `ACTIVE` / `AT_RISK` / `COMPLETED` |
| **نوع المصدر** | طريقة إنتاج القيمة (انظر أدناه) |
| **نوع الدورية** | تكرار القياس: `MONTHLY` شهري / `QUARTERLY` ربع سنوي / `YEARLY` سنوي |
| **الوحدة** | وحدة العرض (مثل: ٪، ريال، عدد) |
| **الاتجاه** | `INCREASE_IS_GOOD` الزيادة مؤشر إيجابي أو `DECREASE_IS_GOOD` الانخفاض مؤشر إيجابي |
| **طريقة التجميع** | كيفية دمج عدة قيم: `LAST_VALUE` / `SUM` / `AVERAGE` / `MIN` / `MAX` |
| **القيمة الأساسية** | قيمة المرجع الابتدائية (الخط الأساسي) |
| **القيمة المستهدفة** | الهدف الرقمي المراد تحقيقه |
| **الوزن** | ترجيح اختياري يُستخدم في حسابات درجة الكيان الأب |
| **الصيغة الحسابية** | تعبير رياضي يُستخدم في أنواع المصدر CALCULATED/DERIVED/SCORE |

---

## أنواع المصدر

| نوع المصدر | المعنى |
|-----------|--------|
| **MANUAL (يدوي)** | يُدخل المستخدم قيمة رقمية يدوياً في كل دورة |
| **CALCULATED (محسوب)** | تُحسب القيمة تلقائياً بواسطة النظام باستخدام صيغة حسابية ومتغيرات مُعرَّفة |
| **DERIVED (مشتق)** | تُستمَد القيمة تلقائياً من قيمة كيان آخر |
| **SCORE (درجة)** | درجة مركّبة مُجمَّعة من الكيانات الفرعية |

---

## إنشاء كيان جديد (للمسؤول فقط)

1. انتقل إلى **الكيانات ← [اسم النوع]**.
2. انقر على زر **جديد** (أو **+**) في الزاوية العلوية.
3. املأ الحقول المطلوبة:
   - **العنوان** (مطلوب)
   - **المالك** (مطلوب)
   - **القيمة المستهدفة** (مطلوبة لتتبع مؤشرات الأداء)
   - **نوع الدورية** (مطلوب)
   - **نوع المصدر** (مطلوب)
4. انقر على **حفظ**.

يظهر الكيان الجديد فوراً في القائمة.

---

## تعديل كيان (للمسؤول فقط)

1. من قائمة الكيانات، انقر على أيقونة **القلم (تعديل)** في الصف، أو افتح صفحة تفاصيل الكيان وانقر على **تعديل**.
2. عدّل الحقول المطلوبة.
3. انقر على **حفظ**.

> **تنبيه مهم:** ينبغي الحذر عند تغيير القيمة المستهدفة أو الصيغة الحسابية لمؤشر أداء نشط — تبقى القيم التاريخية كما هي، لكن حسابات الإنجاز المستقبلية ستستخدم الهدف/الصيغة الجديدة.

---

## حذف كيان (للمسؤول فقط)

1. من قائمة الكيانات، انقر على أيقونة **سلة المهملات** في الصف.
2. تظهر نافذة تأكيد.
3. أكّد الحذف.

الحذف **حذف ناعم (Soft Delete)** — يُخفى الكيان من جميع القوائم لكنه يظل محفوظاً في قاعدة البيانات لأغراض التدقيق. لا يمكن التراجع عن هذا الإجراء من الواجهة.

---

## متغيرات الكيان (لمؤشرات الأداء المحسوبة)

بالنسبة للكيانات ذات `sourceType = CALCULATED`، تُعرَّف **متغيرات** تُغذّي الصيغة الحسابية:

| حقل المتغير | الوصف |
|------------|-------|
| **الرمز (Code)** | معرّف قصير يُستخدم في الصيغة (مثل: `revenue`، `cost`) |
| **اسم العرض** | تسمية مقروءة للمستخدم |
| **نوع البيانات** | `NUMBER` رقم أو `PERCENTAGE` نسبة مئوية |
| **مطلوب** | هل يجب تضمين هذا المتغير عند إدخال قيمة؟ |
| **ثابت** | إذا كان محدداً، يستخدم قيمة ثابتة بدلاً من الإدخال لكل دورة |
| **القيمة الثابتة** | القيمة المستخدمة عندما يكون `isStatic = true` |

عند إدخال قيمة لكيان محسوب، تملأ جميع متغيرات الإدخال غير الثابتة، ويحسب النظام نتيجة الصيغة تلقائياً.

---

## التكليفات

يمكن **تكليف** المستخدمين بكيان (عبر `UserEntityAssignment`). المستخدمون المكلَّفون:
- يمكنهم رؤية الكيان حتى لو لم يكن بإمكانهم ذلك بطريقة أخرى
- يظهرون في تبويب **التكليفات** في صفحة تفاصيل الكيان

تُدار التكليفات من قِبَل المسؤول من صفحة تفاصيل الكيان.

---

## المرفقات

يمكن إرفاق ملفات أو روابط بأي كيان:

| نوع المرفق | الوصف |
|-----------|-------|
| **ملف (FILE)** | ملف مُحمَّل (يُخزَّن في مساحة التخزين السحابي) |
| **رابط (URL)** | رابط لمورد خارجي |

تظهر المرفقات في تبويب **المرفقات** ويمكن إضافتها من قِبَل أي مستخدم يملك صلاحية الكتابة على الكيان.
