# حالات الاستخدام ورحلات المستخدمين — رافد KPI

<div dir="rtl">

## هيكل حالة الاستخدام

تتبع كل حالة استخدام هذا الهيكل:

```
## ملخص حالة الاستخدام    — المعرّف، الأطراف، المحفّز، الشروط المسبقة واللاحقة
## رحلة المستخدم           — مخطط انسيابي مرئي للرحلة من البداية إلى النهاية
## التدفق الرئيسي          — شرح تفصيلي مرقّم خطوةً بخطوة
## التدفقات البديلة        — الحالات الاستثنائية والمسارات المتغايرة
## القواعد التجارية        — القيود والمنطق الذي يحكم حالة الاستخدام
## الشاشات المعنية         — أسماء الصفحات وأنماط الروابط
```

## ملخص الأطراف الفاعلة

| الطرف الفاعل | الدور | حالات الاستخدام النموذجية |
|-------------|-------|--------------------------|
| **المدير (Manager)** | يُدخل بيانات مؤشرات الأداء؛ يتابع محفظته | UC-01، UC-07 |
| **التنفيذي (Executive)** | يراجع الأداء؛ يعتمد القيم (إذا هُيِّئ ذلك) | UC-02، UC-04 |
| **مسؤول المؤسسة (Admin)** | يدير مؤشرات الأداء والمستخدمين وإعدادات المؤسسة | UC-03، UC-04، UC-05 |
| **المعتمِد (Approver)** | يراجع القيم المُرسَلة ويعتمدها أو يرفضها | UC-04 |
| **مسؤول المنصة (Super Admin)** | يُنشئ المؤسسات ويديرها على مستوى المنصة | UC-06 |
