# UC-06 — مسؤول المنصة ينشئ مؤسسة جديدة

```

      │
      ▼

      │
      ▼

      │
      ▼

      │

      ▼

      │  (مدير / تنفيذي / مسؤول)
      ▼
يُهيّئ أنواع الكيانات
      │  الأنواع الافتراضية مُعبَّأة مسبقاً: ركائز، أهداف،
      │  إدارات، مبادرات، مؤشرات أداء
      │  يضيف / يحذف / يُعيد الترتيب حسب الحاجة
      ▼

      │  مطلوب مسؤول واحد على الأقل
      │  يملأ: الاسم، البريد الإلكتروني، كلمة المرور، الدور
      ▼

      │
      ▼

      │

      │

      ▼
```

## التدفق الرئيسي (خطوةً بخطوة)

1. يسجّل مسؤول المنصة دخوله. يُظهر الشريط الجانبي قسم **مسؤول المنصة** (مرئي فقط لدور `SUPER_ADMIN`).
2. ينتقل مسؤول المنصة إلى **مسؤول المنصة ← المؤسسات** (`/<locale>/super-admin/organizations`).
3. تُحمَّل قائمة المؤسسات وتعرض جميع المؤسسات الموجودة.
4. ينقر مسؤول المنصة على **+ مؤسسة جديدة**.
5. يفتح نموذج **إنشاء مؤسسة** (`/<locale>/super-admin/organizations/create`).
6. يملأ مسؤول المنصة **هوية المؤسسة**:
   - **الاسم** (بالإنجليزية) — مطلوب، مثل: "Al-Mosa Group"
   - **الاسم** (بالعربية) — مثل: "مجموعة الموسى"
   - **النطاق (Domain)** — نطاق البريد الإلكتروني لتحديد نطاق المستخدمين، مثل: `almousa.com.sa`
   - **رابط الشعار** — رابط اختياري لصورة شعار المؤسسة
   - **الرسالة** (بالإنجليزية والعربية) — رسالة المؤسسة
   - **الرؤية** (بالإنجليزية والعربية) — رؤية المؤسسة
   - **النبذة** (بالإنجليزية والعربية) — وصف مختصر للمؤسسة
7. يضبط مسؤول المنصة **مستوى اعتماد مؤشرات الأداء** — الدور المخوَّل باعتماد القيم المُرسَلة:
   - `MANAGER` — يعتمد المدير
   - `EXECUTIVE` — يعتمد التنفيذي
   - `ADMIN` — يعتمد مسؤول المؤسسة
8. يراجع مسؤول المنصة ويُهيّئ **أنواع الكيانات**. النموذج مُعبَّأ مسبقاً بـ 5 أنواع افتراضية:

   | الرمز | الاسم (إنجليزي) | الاسم (عربي) |
   |-------|----------------|-------------|
   | `pillar` | Pillars | الركائز |
   | `objective` | Objectives | الأهداف |
   | `department` | Departments | الإدارات |
   | `initiative` | Initiatives | المبادرات |
   | `kpi` | KPIs | مؤشرات الأداء |

   يمكن لمسؤول المنصة: إضافة أنواع جديدة، حذف غير المطلوب منها، إعادة الترتيب بأزرار الأعلى/الأسفل، وتعديل الأسماء.

9. يُضيف مسؤول المنصة **المستخدمين الأوليين** (مطلوب مسؤول واحد على الأقل بدور `ADMIN`):
   - **الاسم** — الاسم الكامل
   - **البريد الإلكتروني** — عنوان بريد إلكتروني فريد
   - **كلمة المرور** — كلمة المرور الأولية
   - **الدور** — `ADMIN` أو `EXECUTIVE` أو `MANAGER`
   - يمكن إضافة مستخدمين إضافيين بزر **+ إضافة مستخدم**.
10. ينقر مسؤول المنصة على **إنشاء المؤسسة**.
11. يتحقق النظام من جميع المدخلات (رموز أنواع الكيانات فريدة، وجود مسؤول واحد على الأقل، الحقول المطلوبة مملوءة).
12. عند النجاح، تُنشأ المؤسسة بصفقة واحدة تشمل جميع أنواع الكيانات والمستخدمين.
13. يُعاد توجيه مسؤول المنصة إلى صفحة تفاصيل المؤسسة الجديدة (`/<locale>/super-admin/organizations/<orgId>`).
14. يراجع مسؤول المنصة المؤسسة المُنشأة وأنواع الكياناتها ومستخدميها.
15. اختيارياً، يُحمّل مسؤول المنصة البيانات الأولية للمؤسسة عبر واجهة التحميل الأولي.
16. يُشارك مسؤول المنصة بيانات اعتماد مستخدم المسؤول بشكل آمن مع المؤسسة العميلة.

## التدفقات البديلة

### البديل أ — فشل التحقق (رمز نوع كيان مكرر)
في الخطوة 11، يكتشف النظام وجود نوعَي كيانات برمز متطابق.
- يظهر خطأ: "رموز أنواع الكيانات يجب أن تكون فريدة."
- يُصحح مسؤول المنصة الرمز المكرر ويُعيد الإرسال.

### البديل ب — لم يُضَف مستخدم بدور مسؤول
في الخطوة 11، يكتشف النظام عدم وجود مستخدم بدور `ADMIN` في القائمة.
- يظهر خطأ: "مطلوب مستخدم مسؤول واحد على الأقل."
- يُضيف مسؤول المنصة مستخدماً بدور مسؤول ويُعيد الإرسال.

### البديل ج — تعديل مؤسسة موجودة
ينتقل مسؤول المنصة إلى قائمة المؤسسات وينقر على صف مؤسسة موجودة.
- تفتح صفحة تفاصيل المؤسسة مع حوارات تعديل لكل قسم: الاسم، النطاق، مستوى الاعتماد، الرسالة/الرؤية/النبذة، أنواع الكيانات، والمستخدمين.
- يمكن تعديل كل قسم بشكل مستقل.

### البديل د — حذف مؤسسة
في صفحة تفاصيل المؤسسة، ينقر مسؤول المنصة على **حذف المؤسسة**.
- يُحذّر حوار تأكيد من أن هذا الإجراء مدمّر.
- عند التأكيد، تُحذف المؤسسة وجميع بياناتها بشكل ناعم.

## القواعد التجارية

- إنشاء المؤسسة **بصفقة واحدة (Atomic)** — إذا فشل أي جزء (مثل بريد إلكتروني مكرر)، يُتراجع عن الإنشاء بالكامل.
- **رموز** أنواع الكيانات يجب أن تكون بأحرف صغيرة، فريدة داخل المؤسسة، وثابتة بعد أن تُنشأ كيانات تُشير إليها.
- مطلوب **مستخدم واحد بدور `ADMIN`** على الأقل عند إنشاء المؤسسة.
- حقل `domain` يُستخدم لتحديد نطاق المستخدمين — المستخدمون ذوو نطاقات البريد الإلكتروني المتطابقة يُربطون بهذه المؤسسة.
- مستخدمو مسؤول المنصة لا يمكنهم الوصول إلى الفضاء الاعتيادي لأي مؤسسة (`/overview`، `/entities`، إلخ) — يعملون فقط في لوحة `/super-admin`.
- عمليات تحميل البيانات الأولية تُلحق البيانات بالمؤسسة أو تستبدلها — تُستخدم فقط أثناء الإعداد الأولي أو لأغراض العروض التجريبية.
