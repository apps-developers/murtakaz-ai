# نظرة عامة على التقنية المستخدمة

<div dir="rtl">

نظرة شاملة على تقنيات منصة  المؤشرات KPI لتنفيذ الاستراتيجية وإدارة الأداء.

---

## المكدس الأساسي (Core Stack)

| التقنية | الاستخدام | الإصدار |
|---------|----------|---------|
| **Next.js** | إطار عمل الويب مع React و App Router | 15.x |
| **React** | مكتبة بناء واجهات المستخدم | 19.x |
| **TypeScript** | لغة البرمجة مع النوعية الصارمة | 5.x |
| **Tailwind CSS** | إطار التصميم والتنسيق | 3.x |
| **Prisma** | ORM وإدارة قاعدة البيانات | 6.x |
| **PostgreSQL** | قاعدة البيانات العلائقية | 15+ |
| **NextAuth.js** | المصادقة والتفويض | 4.x |
| **i18n** | التدويل والترجمة | مدمج |

---

## لماذا هذا المكدس؟

### Next.js + TypeScript
- تطوير full-stack بكفاءة عالية
- توجيه يعتمد على نظام الملفات (File-system routing)
- Server Actions للتعديلات الآمنة
- التحقق من النوعية القوية (Strong typing)

### Tailwind CSS
- تكرار سريع لواجهة المستخدم
- رموز تصميم متناسقة (Design tokens) للألوان والمسافات والطباعة
- تخصيص سهل عبر tailwind.config.js

### PostgreSQL
- سلامة العلاقات بين البيانات (الاستراتيجية ← المبادرة ← المشروع ← مؤشر الأداء)
- دعم التدقيق والمراجعة (Audit trails)
- أداء عالي للاستعلامات المعقدة

### Prisma
- استعلامات آمنة النوعية (Type-safe queries)
- إدارة المخطط ككود (Schema-as-code)
- ترحيلات مرتبطة بتاريخ Git

### NextAuth.js
- مصادقة مجربة ومختبرة
- دعم مزودي الهوية المتعددين (SSO/OAuth)
- إدارة الجلسات بمرونة

### i18n
- دعم اللغتين العربية والإنجليزية
- دعم RTL للعربية و LTR للإنجليزية
- تنسيق التواريخ والأرقام حسب اللغة

## المكونات الرئيسية

### 1. الواجهة الأمامية (Frontend/UI)

- **Next.js App Router**: الصفحات في `web/src/app/[locale]/*`
- **Tailwind CSS**: التصميم والتنسيق
- **RBAC**: التحقق من الأدوار على مستوى المكونات
- **RTL/LTR**: دعم الاتجاهين للغتين

### 2. التدويل (i18n)

- **Locale routing**: التوجيه عبر `/ar` و `/en`
- **Translation dictionaries**: ملفات الترجمة في `web/messages/`
- **RTL handling**: معالجة الاتجاه من اليمين لليسار للعربية

### 3. API والخادم

- **Next.js Route Handlers**: معالجات الطلبات
- **Server Actions**: إجراءات الخادم للتعديلات الآمنة
- **Zod**: التحقق من صحة الإدخالات

### 4. قاعدة البيانات

- **PostgreSQL**: قاعدة البيانات الرئيسية
- **Prisma ORM**: إدارة قاعدة البيانات
- **Migrations**: الترحيلات باستخدام `prisma migrate`

### 5. المصادقة (Auth)

- **NextAuth.js**: إدارة المصادقة
- **JWT**: استراتيجية الجلسات الافتراضية
- **RBAC**: التحكم في الوصول المستند إلى الأدوار

## الأمان والامتثال

- HTTPS إلزامي في الإنتاج
- Cookies آمنة للمصادقة
- RBAC على جانب الخادم فقط
- لا تثق بالأدوار من العميل
- سجلات التدقيق (Audit logs) لجميع التغييرات
- النسخ الاحتياطي لقاعدة البيانات
- الاحتفاظ بالبيانات ≥ 24 شهراً
